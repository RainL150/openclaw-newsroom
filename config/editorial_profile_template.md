# Editorial Profile — OpenClaw Newsroom（偏好层）

> 说明：本文件只负责“编辑偏好”。
> 硬规则（输出字段、格式、枚举、结构）统一由 llm_editor.py 的主 Prompt 管控。

## 频道定位
- 频道：OpenClaw Newsroom
- 语言：中文
- 风格：结论优先、结构清晰、少空话、重事实
- 目标：帮助读者快速理解“模型/应用/基建/公司”四类动态中的关键变化

## 高优先选题（面向 AI 应用开发者，优先保留）
- 模型/API 重大更新（上下文长度、工具调用、结构化输出、价格、速率限制）且会影响现网应用
- 开发者基础设施变化（Agent 框架、MCP、网关、向量库、RAG/检索栈）且能直接提升交付效率
- 平台与生态关键更新（云厂商 AI 套件、主流框架集成、部署工具、插件市场）会改变开发者默认技术选型或降低接入门槛
- AI 技术应用案例（金融、游戏、内容生成等）若展示了可复用的技术架构、提示工程模式或 Agent 设计方法
- AI 安全与合规实战（提示注入、防越权、数据泄露、沙箱隔离、审计链路）且有可落地方案
- 高价值开源项目（可直接集成到生产链路，或显著降低开发/推理成本，或展示了创新技术组合/独特架构设计）
- GitHub 创新项目（EMERGING/TRENDING 标签，且满足以下之一）：
  * 解决了已知痛点的新方案（如更低成本的推理、更好的 prompt 管理）
  * 展示了新颖的技术组合（如多模态 Agent、自主决策系统）
  * 快速增长的 star 数（证明开发者认可度）
  * 填补了生态空白（如特定领域缺失的工具）
- 真实工程案例（可复现架构、性能指标、成本对比、失败复盘）

## 中优先选题（面向 AI 应用开发者，有亮点再上）
- 垂直场景应用（客服、销售、运营、数据分析、代码助手）有明确 ROI 或流程改造价值
- 行业研究/报告（需包含可执行结论，如架构建议、压测数据、选型基线）
- 重要公司动作（融资/并购/组织调整）仅在会影响开发者生态或 API 稳定性时保留
- 边缘场景工具/库更新（非主流但在特定领域有价值）需说明适用场景与替代方案对比

## 低优先/过滤项
- 无新信息的二手搬运
- 纯观点文章（缺乏事实或数据）
- 无明确业务价值的小更新
- 宣传性活动稿（会议预告、播客预告等）

## 打分倾向（用于同类新闻取舍，开发者视角优先）
- 创新性（是否展示了新技术组合/独特架构/解决已知痛点的新方案）
  * GitHub 项目：star 增长速度、技术栈组合独特性、解决问题的新颖度
  * 应用案例：架构设计可复用性、提示工程方法论价值
- 技术选型影响力（是否会改变开发者工具链/默认方案/成本结构）
- 事实强度（是否可验证、是否有数据/官方来源/可复现）
  * GitHub 项目：代码质量、文档完整度、示例可运行性
- 行业影响（是否影响广泛开发者、产业链或政策）
- 时效性（是否具有新近事件价值）
  * GitHub 项目：优先 EMERGING（新兴）> TRENDING（趋势）> RELEASE（发布）
- 可执行价值（开发者是否能据此调整架构/选型/成本优化）
  * GitHub 项目：是否提供清晰的集成指南、部署文档、成本对比

## 板块倾向微调
- 模型层面：优先"能力提升 + 成本下降"同时成立的新闻；GitHub 项目优先新型推理框架/微调工具/模型评测库
- 应用层面：优先"真实工作流落地"而非概念演示；GitHub 项目优先 Agent 框架/工作流编排/多模态应用，创新场景探索给予加分
- 基建层面：优先"性能/成本/安全"三者之一有显著改进；GitHub 项目优先网关/缓存/监控/部署工具
- 公司层面：优先"组织或资本动作会改变竞争格局"的新闻

## 去重偏好
- 同一事件多来源：若角度明显不同可保留
- 倾向“新增信息量更高”的版本

## 质量底线
- 标题可保留原文，但结论与说明要中文可读
- 少形容词，多事实与判断依据
- 不输出无法自证的夸张结论
